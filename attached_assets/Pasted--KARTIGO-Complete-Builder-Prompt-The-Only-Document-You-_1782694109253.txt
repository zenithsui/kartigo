## 🧠 BEFORE YOU WRITE A SINGLE LINE OF CODE — READ THIS

You are building **Kartigo** — a full-stack, production-ready, launch-grade e-commerce platform that blends the DNA of four Indian e-commerce giants into one cohesive experience:

| Platform | What Kartigo takes from it |
|----------|---------------------------|
| **Amazon** | Catalog depth, verified reviews with photos, trust badges, delivery estimates, seller marketplace infrastructure |
| **Flipkart** | Flash sales with countdown timers, Big Billion Day-style banners, SuperCoins-style reward system (KartiCoins), deal-driven excitement |
| **Meesho** | Reseller share-and-earn model, social commerce links, referral tracking, commission-based reselling, low-price focus |
| **Myntra** | Fashion-forward UI, brand storefronts, lookbook-style curated collections, size/color/style filters, premium design aesthetic |

**These are not four feature sets bolted together.** They are one unified shopping experience where a user can:

1. Browse curated fashion lookbooks styled like Myntra →
2. Discover a flash sale with a live countdown timer like Flipkart →
3. Read 47 verified purchase reviews with photo evidence like Amazon →
4. Share the product to their WhatsApp group and earn commission like Meesho →
5. Checkout with Razorpay and earn KartiCoins →
6. Track their order with a real-time status timeline

**That's one user journey. That's Kartigo.**

---

### 🚫 THE RULES — NON-NEGOTIABLE

1. **Every feature works end-to-end.** If a button exists, it does something real. If a page exists, it's fully built. If a flow exists, it completes. No placeholders. No "coming soon." No `TODO` comments. No `lorem ipsum`. No empty modals. No skeleton screens that never resolve.

2. **This ships tomorrow.** Write every component, every API route, every database query as if a real Indian consumer will use it within 24 hours.

3. **Build for India.** Prices in ₹ (INR). Addresses with PIN codes. Phone numbers with +91. Payment via Razorpay. Delivery estimates in Indian geography. Hindi/English sensibility in UX copy.

4. **No generic Bootstrap/template aesthetics.** The homepage must feel alive — like walking into a massive mall during Diwali sale season. If it looks like a tutorial project, you've failed.

5. **Zero `any` types in TypeScript.** Strict mode. Type everything. Prisma-generated types flow from DB → API → Frontend.

6. **No hardcoded secrets.** Everything through environment variables.

### What Failure Looks Like

- A homepage that looks like a Bootstrap starter template
- Product pages without reviews, ratings, delivery estimates, or seller info
- A cart that doesn't persist across sessions
- An admin panel that can't actually manage products
- Auth that's half-implemented (login works, register doesn't)
- Search that only matches exact titles
- Mobile experience that's an afterthought
- Flash sale timers that don't count down
- "Share & Earn" buttons that generate no link

**If any of the above describes what you build, you have failed. Don't let it happen.**

// prisma/schema.prisma

generator client {

// ═══════════════════════════════════════

// ═══════════════════════════════════════

model User {
  id              String    @id @default(cuid())
  name            String
  email           String    @unique
  phone           String?   @unique
  passwordHash    String?
  avatar          String?
  role            UserRole  @default(CUSTOMER)
  emailVerified   DateTime?
  phoneVerified   Boolean   @default(false)
  referralCode    String    @unique @default(cuid())
  referredBy      String?

  // Relations
  addresses       Address[]
  orders          Order[]
  reviews         Review[]
  wishlistItems   WishlistItem[]
  cart            Cart?
  seller          Seller?
  referralsMade   Referral[]       @relation("ReferralsMade")
  referralReceived Referral?       @relation("ReferralReceived")
  shareLinks      ShareLink[]
  rewardHistory   RewardTransaction[]
  notifications   Notification[]

  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

model Address {

- **Razorpay (default):** "Pay with Razorpay" button

    1. POST `/api/payment/create-order` → creates Razorpay order on server
    2. Opens Razorpay checkout popup on frontend (using `razorpay` JS SDK)
    3. User completes payment in popup
    4. On success callback → POST `/api/payment/verify` with razorpayOrderId, razorpayPaymentId, razorpaySignature
    5. Server verifies signature using `crypto.createHmac`
    6. On valid → create Order in database, clear cart, credit KartiCoins, redirect to confirmation
    7. On invalid → show error, allow retry

- **Razorpay configuration (lib/razorpay.ts):**
  ```typescript
  // Server-side Razorpay instance
  import Razorpay from 'razorpay'
  const razorpay = new Razorpay({
    key_id: process.env.RAZORPAY_KEY_ID!,
    key_secret: process.env.RAZORPAY_KEY_SECRET!,
  })
  ```
- **Razorpay popup options:**
  ```typescript
  {
    key: process.env.NEXT_PUBLIC_RAZORPAY_KEY_ID,
    amount: totalInPaise, // amount in paise (₹100 = 10000)
    currency: "INR",
    name: "Kartigo",
    description: `Order #${orderNumber}`,
    order_id: razorpayOrderId,
    handler: async (response) => { /* verify */ },
    prefill: { name, email, contact },
    theme: { color: "#3730A3" }
  }
  ```
- **Webhook handler (`/api/payment/webhook`):**
  - Verify webhook signature using RAZORPAY_WEBHOOK_SECRET
  - Handle events: `payment.captured`, `payment.failed`, `refund.processed`
