# 🚀 KARTIGO — The Build Prompt

> **Read this entire document before writing a single line of code. Every sentence exists for a reason. Every detail is intentional. This is not a wishlist — it is a blueprint.**

---

## SECTION 0 — MINDSET & NON-NEGOTIABLES

You are about to build **Kartigo** — India's most complete shopping platform. Not a clone. Not a template. Not a portfolio piece. A *real product* that fuses the DNA of **Amazon, Flipkart, Meesho, and Ajio** into a single, cohesive, production-grade e-commerce experience.

### The Standard You Must Hold

- **Every feature you build must work end-to-end.** If a button exists, it does something. If a page exists, it's fully functional. If a flow exists, it completes. No placeholders. No "coming soon." No skeleton screens that never load real data. No `TODO` comments. No `lorem ipsum`. No empty modals.
- **This ships tomorrow.** Write every component, every API route, every database query as if a real user will interact with it within 24 hours. Because that's the plan.
- **Quality is not optional.** If you find yourself about to cut a corner — stop. Re-read this section. Then build it properly.
- **You are building for India.** Prices in ₹ (INR). Addresses with PIN codes. Phone numbers with +91. Payment via Razorpay. Delivery estimates in Indian geography context. Hindi/English sensibility in UX copy.

### What Failure Looks Like

- A homepage that looks like a Bootstrap template
- Product pages without reviews, ratings, delivery estimates, or seller info
- A cart that doesn't persist across sessions
- An admin panel that can't actually manage products
- Authentication that's half-implemented
- Search that only matches exact titles
- Mobile experience that's an afterthought

**If any of the above describes what you build, you have failed. Do not let that happen.**

## SECTION 1 — PRODUCT VISION

**Kartigo** = Amazon's catalog depth & trust infrastructure + Flipkart's deal-driven excitement & gamification + Meesho's social commerce & reseller economics + Ajio's fashion-forward design & brand storytelling.

These are not four separate feature sets bolted together. They are **one unified experience** where:

- A user can browse curated fashion lookbooks (Ajio) →
- Discover a flash sale with countdown timer (Flipkart) →
- Read 47 verified reviews with photo evidence (Amazon) →
- Share the product with their WhatsApp group to earn commission (Meesho) →
- Checkout with Razorpay and earn reward coins (Flipkart SuperCoins) →
- Track their order with real-time status updates (Amazon)

**That's one user journey. That's Kartigo.**

## SECTION 2 — TECHNOLOGY STACK

### Core Framework
```
Framework:        Next.js 14+ (App Router)
Language:         TypeScript (strict mode, no `any` types)
Styling:          Tailwind CSS 3.4+
UI Components:    shadcn/ui as the base component library
State Management: Zustand for client state, React Query (TanStack Query) for server state
Forms:            React Hook Form + Zod validation
```

### Backend & Data
```
Database:         PostgreSQL via Supabase (or Neon)
ORM:              Prisma
Authentication:   NextAuth.js v5 (Auth.js) with Credentials + Google + Phone OTP
File Storage:     Supabase Storage or Cloudinary for product images
Search:           Implement full-text search via PostgreSQL tsvector or integrate Meilisearch
Payments:         Razorpay (test mode keys, but fully integrated flow)
Email:            Resend or Nodemailer for transactional emails
```

### Deployment & Infrastructure
```
Hosting:          Vercel-ready (next.config optimized for Vercel deployment)
Environment:      .env.example with every required variable documented
Edge Functions:   Use Next.js middleware for auth guards, geo-detection
Image CDN:        Next.js <Image> with Cloudinary/Supabase loader
```

### Code Quality
```
Linting:          ESLint with strict Next.js config
Formatting:       Prettier
Type Safety:      End-to-end type safety from DB → API → Frontend via Prisma generated types
File Structure:   Feature-based organization inside /app directory
```

## SECTION 5 — FEATURE SPECIFICATIONS

### 5.1 — Authentication & Authorization

```
- NextAuth.js v5 with:
  - Credentials provider (email/password with bcrypt hashing)
  - Google OAuth provider
  - Phone OTP provider (mock OTP for now — use a fixed code like 123456 in dev)
- Role-based access control (RBAC):
  - BUYER: default role, can browse/purchase/review
  - SELLER: can manage products, view orders for their items
  - ADMIN: full platform access
- Protected routes:
  - /account/* → requires BUYER+
  - /seller/* → requires SELLER
  - /admin/* → requires ADMIN
  - /checkout → requires BUYER+
- Session persistence across tabs
- JWT tokens with secure httpOnly cookies
- Middleware-based route protection
```

### 5.2 — Search

```
- Full-text search using PostgreSQL tsvector/tsquery OR Meilisearch
- Search autocomplete dropdown:
  - Product name matches (with thumbnail + price)
  - Category matches
  - Brand matches
  - Recent searches (stored in localStorage)
  - Trending searches (hardcoded or from analytics)
- Debounced input (300ms)
- Search results page with full filtering capability
- URL-based search state (querystring) for shareability
- Search analytics tracking (what was searched, clicked)
```

### 5.3 — Cart System

```
- Persistent cart:
  - Authenticated users: stored in database
  - Guest users: stored in localStorage, merged on login
- Cart operations:
  - Add item (with variant selection)
  - Update quantity (respect stock limits)
  - Remove item
  - Move to wishlist
  - Apply coupon code
  - Apply/toggle KartiCoins
- Cart badge count in header (real-time)
- Mini cart preview on hover (desktop)
- Price recalculation on every change
- Stock validation before checkout
```

### 5.4 — Payment Integration (Razorpay)

```
- Full Razorpay integration:
  1. Create Razorpay order (API route: POST /api/payment/create-order)
  2. Open Razorpay checkout popup on frontend
  3. Handle success callback → verify signature (API route: POST /api/payment/verify)
  4. Update order status on success
  5. Handle failure → show error, allow retry
- Use Razorpay TEST keys (key_id and key_secret from Razorpay test dashboard)
- Support for:
  - Cards (Visa, Mastercard, etc.)
  - UPI
  - Net Banking
  - Wallets (Paytm, PhonePe, etc.)
- Cash on Delivery option (toggle, with COD fee)
- Payment status webhook handling
- Refund flow (admin-triggered)
```

### 5.5 — KartiCoins Reward System (SuperCoins-style)

```
- Earn rate: 5 KartiCoins per ₹100 spent (configurable by admin)
- Redemption rate: 1 KartiCoin = ₹1 discount (configurable)
- Earn on: completed orders only (coins credited after delivery)
- Bonus coins: sign-up bonus (50 coins), referral bonus (100 coins), review bonus (10 coins)
- Expiry: coins expire after 12 months (track expiry date)
- Transaction history with type, amount, description, date
- Balance display in header (for logged-in users) and rewards page
- Apply coins at checkout (partial payment)
```

### 5.6 — Social Commerce / Reseller System (Meesho-style)

```
- Any user can become a reseller (toggle in account settings)
- Reseller flow:
  1. Browse any product
  2. Click "Share & Earn" button
  3. Generates unique tracked link: kartigo.com/product/slug?ref=USER_CODE
  4. Share via WhatsApp, Instagram, Facebook, copy link
  5. When someone purchases through this link:
     - Referrer earns commission (% of sale, configurable per category)
     - Commission tracked in reseller dashboard
- Share link tracking:
  - Clicks, unique visitors, add-to-carts, purchases
- Reseller dashboard with:
  - Total earnings (lifetime, this month)
  - Product-wise performance
  - Withdrawal history (mock — show "Withdrawal requested" state)
- WhatsApp deep link sharing: "Check out this amazing deal on Kartigo! ₹1,299 only! [link]"
```

### 5.7 — Referral System

```
- Every user gets a unique referral code on signup
- Share referral code/link → friend signs up → both earn KartiCoins
- Referrer: 100 coins on friend's first purchase
- Referee: 50 coins on signup
- Referral tracking: invited list, signup status, reward status
- Referral link format: kartigo.com/?ref=CODE
```

### 5.8 — Flash Sales & Promotions

```
- Flash sale creation (admin):
  - Select products, set flash sale prices
  - Set start and end time
  - Set quantity limit per user
- Frontend display:
  - Countdown timer (live, real-time)
  - "X% claimed" progress bar
  - "Hurry! Only Y left" urgency text
  - Products show original price vs flash price
- Auto-start and auto-end based on time
- Sale-specific banners on homepage

- Coupons:
  - Percentage or fixed discount
  - Minimum order value
  - Maximum discount cap
  - Usage limit (total and per-user)
  - Date-based validity
  - Category/product restrictions
  - Apply at cart/checkout with validation
```

### 5.9 — Reviews & Ratings

```
- Only verified purchasers can review (check order history)
- Review form:
  - Star rating (1-5, required)
  - Title (optional)
  - Body text (required, min 20 characters)
  - Photo upload (up to 5 images)
- Review display:
  - Aggregate rating with breakdown bar chart
  - Individual reviews with user name, date, verified badge
  - "Helpful" voting
  - Photo reviews shown prominently
- Admin moderation:
  - Reported reviews queue
  - Approve/reject
- Seller response capability
```

### 5.10 — Multi-Vendor / Marketplace

```
- Seller onboarding:
  - Application form (store name, GST, PAN, bank details)
  - Admin approval workflow
- Each product belongs to a seller
- Orders split by seller (one order can have items from multiple sellers)
- Each seller manages their own:
  - Products (CRUD with approval workflow)
  - Orders (their items only)
  - Revenue and payouts
- Commission system:
  - Platform takes X% commission (configurable per category)
  - Seller receives (100-X)% after delivery
  - Admin manages commission rates
- Seller storefront: public page showing seller info + their products
- Seller ratings: aggregate from their product reviews
```
